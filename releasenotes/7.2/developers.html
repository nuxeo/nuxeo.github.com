7.2 Release Notes

For Developers

JDK 1.8+ is required (version 8u25 minimum)

Java 8 update 25+ is required for building Nuxeo and the constraint has been set as an enforcement in the root Nuxeo POM org.nuxeo:nuxeo-ecm .
Some products like the Automation Client will still be compiled with Java 7 compliance (source and target) for being usable in Java 7 runtime environments but Java 8 is still required to perform their build.

Picture Document Type

Schemas and Layouts Updates

- The following documentation provides the new default contribution for binaries metadata extraction in Nuxeo.

- Exif mapping remains identical.

- Widget "summary_picture_iptc" has been removed from document summary.

- Mistral engine is removed. Use Binary Metadata feature to extract metadata with custom engine. (default one is exiftool since 7.1)

- IPTC metadata are extracted and stored on dublincore schema: the iptc schema is no more used. We should map a limited number of metadata: source, copyrights ( > dc:rights ), legend (> dc:description ).

Exports and Renditions

Rendition definitions can now also be contributed through a class implementing the inferface 

RenditionDefinitionProvider
. When asked for the list of rendition definitions for a document, all matching RenditionDefinitionProviders are called to compute dynamic rendition definitions

.

Video transcoded videos are exposed as renditions (but not visible in the UI).

Picture conversions are also exposed as renditions (but not visible in the UI).

The default exports (XML export, ZIP Tree Export, ...) has been migrated to be exposed as standard renditions, thanks to the new
