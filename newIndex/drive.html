Nuxeo Drive

Nuxeo Drive handles all types of content, from simple files such as documents, images, and slide decks to complex folder hierarchies generated by local applications. It even manages dynamic document lists. You can configure it to synchronize documents that are in a specified state, to handle custom use cases.

Nuxeo Drive Roadmap

  • Application update doesn't always work under Windowsmode_edit

    blablablablabl

  • Use the generic JSP to acquire a token mode_edit

    Taking advantage of NXP-19752, and falling back to the current system using drive_login.jsp if acquire_token.jsp is unavailable.


  • File System Decoration mode_edit

    Decoration of the file system so as to represent the state of the documents.

  • Push notifications for Nuxeo Drive and Mobilemode_edit

    Nuxeo offers a notification service to its customers. Notifications are delivered via Mobile Applications and Nuxeo Drive and can be used for various use cases:
    - tasks notifications
    - collaborative information (someone working on someone else document, comments, ...)
    - real time updates.

  • Nuxeo Drive: Upload Resumemode_edit

    Drive is able to resume the upload of a binary content. This option is interesting in case of big files or bad network conditions.


  • Nuxeo Drive: Dataviz dashboard / monitoring mode_edit

    Dataviz dashboard / monitoring to know:
    - Who downloads/uploads how much documents
    - Real time view of synchronisations

  • Nuxeo Drive: Implement some bitrate controlmode_edit

    Implement some kind of control /limit for downloading / uploading synchronisations.

  • Nuxeo Drive Importermode_edit

    A third feature (after documents syncing and live edit) is added to Nuxeo Drive: Document Import. User can easily benefit from a robust and scalable importer.
